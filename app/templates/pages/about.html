About

How does the app work ?

The user uploads a CMIF file of Alexander von Humboldt letters. The tool performs a query on the Kalliope-Verbund API and compares the user's data to it.

When a full match is found then the Kalliope's link is added to the user's CMIF file in the correspDesc/@corresp. Possible matches are then proposed to the user who can then confirm or not these matches so that they can be added to the file.

The user has the possibility to download in CMIF format the uploaded file with added Kalliope's links or only a file containing the complete matches.

What is a match ?

The tool compares the data on the sender's GND ID and the receiver's GND ID, the coverage place and the sending date. A complete match means that all four elements match. If it is a possible match, i.e. one that needs user evaluation, then all elements match except the coverage place.

CMIF: the accepted format

The Correspondence Metadata Interchange Format (CMIF) was developed for editors of digital and printed scholarly editions to provide the most important metadata about edited letters online and in a machine-readable form. CMIF is based on the guidelines of the Text Encoding Initiative (TEI) and was developed within the TEI Correspondence Special Interest Group (SIG). The format is used on correspSearch.

To know more about CMIF format, please read this article. There are also multiple informations on correspSearch.
